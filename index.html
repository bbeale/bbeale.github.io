Security Engineering

I am highly motivated by studying the ways in which software behavior can differ from what the end user might expect.

While that may be encapsulated by my QE work, testing webapps for security vulnerabilities kicks it up several notches when compared to looking for functional defects due to the methodologies a security tester must use. While functional testing often simulates the role of an end user to interact accordingly with the system under test, security testing involves abusing functionality to make it behave in unexpected and unsafe ways. Because of this, security tests typically do not limit the scope in the same way as functional tests, with some projects even allowing the use of special tools such as automated scanners to speed up the process.

Among my vulnerability findings include stored XSS, SQL injections, and a particularly interesting multipart session vulnerability in a learning management system. However, along with discovering security flaws, I am also comfortable taking off the white hat and putting on the developer hat to mitigate vulnerability disclosed by others. Some of my recent fixes include NPM modules used in many projects.
